# 13.01 Ternary Operators

The **ternary operator** is a short way to write an `if / else`.

---

## Basic syntax

| Part      | Meaning        |
| --------- | -------------- |
| condition | what you check |
| `?`       | if true        |
| `:`       | if false       |

```ts
condition ? valueIfTrue : valueIfFalse;
```

---

## Simple examples

| If / else                       | Ternary equivalent         |
| ------------------------------- | -------------------------- |
| `if (isOpen) show()`            | `isOpen ? show() : hide()` |
| `if (a > b) return a`           | `a > b ? a : b`            |
| `if (loggedIn) "Yes" else "No"` | `loggedIn ? "Yes" : "No"`  |

---

## Using ternary in JSX 

### Conditional rendering

```tsx
{
  isLoading ? <Loader /> : <Content />;
}
```

---

### Show / hide element

```tsx
{
  isAdmin ? <AdminPanel /> : null;
}
```

---

## Assigning values

```ts
const status = isOnline ? "online" : "offline";
```

```ts
const buttonText = disabled ? "Disabled" : "Submit";
```

---

## Nested ternary (use carefully)

```ts
const label = isLoading ? "Loading..." : isError ? "Error" : "Success";
```

Rule:

- OK for **2–3 cases**
- Avoid deep nesting

---

## When ternary is a good idea

- simple conditions
- value assignment
- JSX rendering
- replacing short `if / else`

---

## When NOT to use ternary

Avoid when:

- logic is complex
- more than 2–3 conditions
- readability suffers

Prefer `if / else` instead.

---

## Common mistakes

| Mistake          | Example                     |
| ---------------- | --------------------------- |
| Too complex      | nested ternaries everywhere |
| Side effects     | calling many functions      |
| Missing `:`      | invalid syntax              |
| Hard to read JSX | logic inside JSX            |

---

## Mental model

- `?` → if true
- `:` → if false
- Use for **expressions**, not statements
- Readability > short code

---

## Summary

- Ternary is a short `if / else`
- Very common in React JSX
- Best for simple logic
- Avoid nesting too much
- If it’s hard to read → don’t use it
