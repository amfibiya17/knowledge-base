# 14.01 Regex Basics

**Regex (Regular Expressions)** are patterns used to **match text**.

They are commonly used for:

- validation (email, phone, password)
- searching text
- replacing text

Think of regex as **rules for what text should look like**.

---

## Core regex building blocks

| Pattern | Meaning                | Example match |
| ------- | ---------------------- | ------------- |
| `abc`   | exact text             | `abc`         |
| `.`     | any character          | `a`, `1`, `!` |
| `\d`    | any digit (0–9)        | `5`           |
| `\w`    | letter, number, or `_` | `a`, `7`, `_` |
| `\s`    | whitespace             | space, tab    |

---

## Quantifiers (how many)

| Pattern | Meaning         | Example            |
| ------- | --------------- | ------------------ |
| `+`     | one or more     | `\d+` → `123`      |
| `*`     | zero or more    | `a*` → `""`, `aaa` |
| `?`     | zero or one     | `a?` → `""`, `a`   |
| `{3}`   | exactly 3       | `\d{3}` → `123`    |
| `{2,5}` | between 2 and 5 | `\d{2,5}`          |

---

## Position anchors

| Pattern | Meaning         | Why it matters  |
| ------- | --------------- | --------------- |
| `^`     | start of string | must start with |
| `$`     | end of string   | must end with   |

Example:

```js
// matches a string made of digits only
^\d+$
```

---

## Character groups

| Pattern | Meaning           | Example    |
| ------- | ----------------- | ---------- |
| `[abc]` | one of these      | `a` or `b` |
| `[a-z]` | lowercase letters | `m`        |
| `[A-Z]` | uppercase letters | `Q`        |
| `[0-9]` | digits            | `7`        |

---

## Alternation (OR)

| Pattern        | Meaning         | Example |
| -------------- | --------------- | ------- |
| `cat|dog`      | cat **or** dog  | `cat`  |
| `(jpg|png)`    | jpg **or** png  | `png`  |

---

## Mental model

- Regex = **pattern**
- String = **input**
- Match = “does the text follow the rules?”

Regex is **strict**, not clever.

---

## Summary

- Regex matches text patterns
- Built from small symbols
- Tables > long explanations
- Best for validation and search
