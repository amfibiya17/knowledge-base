# 5.10 CSS Grid

CSS Grid is a CSS layout system for arranging items in **two dimensions**:
**rows AND columns**.

Use it for:

- page layouts
- card grids
- galleries
- dashboards
- complex responsive layouts

---

## Mental model

- **grid container** = the parent (`display: grid`)
- **grid items** = the children
- Grid is best for **2D layout** (rows + columns)
- Flexbox is best for **1D layout** (row OR column)

---

## Create a grid container

```css
.grid {
  display: grid;
}
```

---

## Columns

```css
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
}
```

- `fr` = “fraction of remaining space”
- `1fr 2fr` means the second column is twice as wide

---

## Rows

```css
.grid {
  display: grid;
  grid-template-rows: auto 1fr auto;
}
```

Useful for page layouts (header / content / footer).

---

## Gap

Use `gap` for spacing between grid items.

```css
.grid {
  display: grid;
  gap: 16px;
}
```

---

## Repeat

```css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}
```

Same as `1fr 1fr 1fr`.

---

## Responsive grid

Classic “cards that wrap automatically”:

```css
.grid {
  display: grid;
  gap: 16px;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
}
```

Meaning:

- try to fit as many columns as possible
- each column is at least `240px`
- otherwise it stretches up to `1fr`

---

## Place items (spanning)

### Span columns

```css
.item {
  grid-column: span 2;
}
```

### Span rows

```css
.item {
  grid-row: span 2;
}
```

---

## Named areas (great for page layout)

### CSS

```css
.layout {
  display: grid;
  gap: 16px;
  grid-template-columns: 240px 1fr;
  grid-template-areas:
    "header header"
    "sidebar content"
    "footer footer";
}

.header {
  grid-area: header;
}
.sidebar {
  grid-area: sidebar;
}
.content {
  grid-area: content;
}
.footer {
  grid-area: footer;
}
```

### HTML

```html
<div class="layout">
  <header class="header">Header</header>
  <aside class="sidebar">Sidebar</aside>
  <main class="content">Content</main>
  <footer class="footer">Footer</footer>
</div>
```

---

## Alignment

### Align items inside their cells

```css
.grid {
  justify-items: start; /* left/right inside cell */
  align-items: start; /* top/bottom inside cell */
}
```

Common values: `start | center | end | stretch`

### Align the whole grid inside the container

```css
.grid {
  justify-content: center;
  align-content: center;
}
```

---

## Common patterns

### 1) Simple card grid

```css
.cards {
  display: grid;
  gap: 16px;
  grid-template-columns: repeat(3, 1fr);
}
```

### 2) Responsive card grid

```css
.cards {
  display: grid;
  gap: 16px;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
}
```

### 3) Two-column layout (sidebar + content)

```css
.page {
  display: grid;
  gap: 24px;
  grid-template-columns: 260px 1fr;
}
```

---

## Common mistakes

- using Grid for a simple one-row alignment (Flexbox is simpler)
- forgetting `minmax()` → columns get too small on mobile
- using margins instead of `gap`
- overcomplicating with spans when `auto-fit` would work

---

## Quick cheat sheet

| Goal                       | Use                             |
| -------------------------- | ------------------------------- |
| create columns             | `grid-template-columns`         |
| spacing between items      | `gap`                           |
| responsive columns (wrap)  | `repeat(auto-fit, minmax(...))` |
| make an item wider         | `grid-column: span n`           |
| define page layout regions | `grid-template-areas`           |
| align items inside cells   | `justify-items` / `align-items` |

---

## Summary

- Grid = **2D layout** (rows + columns)
- Parent controls layout with `display: grid`
- `grid-template-columns` defines columns
- `gap` handles spacing cleanly
- `auto-fit + minmax()` is the go-to responsive pattern
- Use Grid for layouts, Flexbox for alignment
