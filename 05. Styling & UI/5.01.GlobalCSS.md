# 5.01 Global CSS

Global CSS means **CSS rules that apply to the whole site**, not just one component.

It usually includes:

- base styles (fonts, colours, background)
- resets / normalisation
- typography defaults
- layout helpers
- design tokens (CSS variables)

---

## What “global” means

Global CSS is loaded once and affects **everything**.

```css
button {
  border-radius: 0;
}
```

Every `<button>` on the site changes.

---

## Why Global CSS exists

Global CSS is used for **shared foundations**, such as:

- app font
- default text colour
- line height
- background colour
- link styles
- spacing rules for headings

It provides a consistent baseline.

---

## Typical things to put in Global CSS

### 1. Reset / normalise

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}

body {
  margin: 0;
}
```

---

### 2. Base typography

```css
body {
  font-family: system-ui, sans-serif;
  line-height: 1.5;
}
```

---

### 3. CSS variables (design tokens)

```css
:root {
  --text: #111;
  --bg: #fff;
  --space-2: 8px;
  --radius: 8px;
}
```

---

### 4. Generic helpers

Examples:

- `.sr-only`
- `.container`

Keep helpers minimal.

---

## What NOT to put in Global CSS

Avoid:

- component-specific styles
- page-specific layouts
- one-off fixes
- random utility classes

If it’s for **one component**, it should not be global.

---

## Global CSS vs Component CSS

**Global CSS**

- site-wide rules
- easy to cause side effects

**Component CSS (CSS Modules)**

- scoped styles
- safer and easier to maintain

---

## Common mistakes

- overly broad selectors
- using global CSS to fix one component
- excessive `!important`
- duplicating values instead of using variables

---

## Mental model

- Global CSS → site rules
- Component CSS → component rules

If changing it could break many pages, keep it minimal.

---

## Summary

- Global CSS affects the entire app
- Best for resets, typography, and design tokens
- Powerful but risky
- Keep it small and stable
- Use component-scoped styles for most UI
